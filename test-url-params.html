React Router URL Parameters Test

Test the following URLs in your browser to verify pagination and search parameters are stored in the URL:

Test 1: Default URL (no parameters)

http://localhost:3001/
Should show page 1 with 10 items per page, no search query.

Test 2: Page 3 with 20 items per page

http://localhost:3001/?page=3&pageSize=20
Should show page 3 with 20 items per page.

Test 3: Search for "john"

http://localhost:3001/?search=john&page=1&pageSize=10
Should search for users with "john" in their name.

Test 4: Combined search and pagination

http://localhost:3001/?search=smith&page=2&pageSize=30
Should show page 2 of search results for "smith" with 30 items per page.

Test 5: Reset to default

http://localhost:3001/?page=1&pageSize=10
Clicking "Reset" button should clear search parameter and reset to page 1.

How to test:

  1. Open the application at http://localhost:3001/
  2. Try changing page or page size - the URL should update automatically
  3. Search for a user - the search term should appear in the URL
  4. Copy a URL with parameters and paste it in a new tab - it should load with the same state
  5. Use browser back/forward buttons - should navigate through previous states

Expected Behavior:

  • URL parameters should update when you change page, page size, or search
  • Bookmarking a URL with parameters should restore the same view
  • Sharing URLs with colleagues should show them the exact same data view
  • Browser history should work correctly with back/forward buttons
